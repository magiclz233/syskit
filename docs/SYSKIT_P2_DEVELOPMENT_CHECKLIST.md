# syskit P2 开发需求清单

## 1. 文档目的

这份文档是 `syskit` P2 阶段的唯一开发控制清单，用于管理以下事项：

1. P2 范围内需要开发的需求项。
2. 需求项的开发顺序、依赖关系和当前状态。
3. 每次开发完成后需要回写的完成标记。
4. 后续开发时默认继续处理的“下一个未完成需求”。

本清单对齐以下文档：

- `docs/SYSKIT_PRODUCT_PRD.md`
- `docs/SYSKIT_CLI_SPEC.md`
- `docs/SYSKIT_ARCHITECTURE.md`

## 2. 使用规则

1. 本文档确认前，不进入正式 P2 开发。
2. 文档确认后，默认严格按本文档自上而下顺序开发。
3. 若某项未完成，默认不跳过到后续项；确需跳过时，需要先明确调整优先级。
4. 每完成一项开发并完成基本验证后，将状态更新为 `已开发`。
5. 下一次继续开发时，从第一个状态不是 `已开发` 的需求项继续。
6. 若某项拆分为多次提交，状态可暂记为 `开发中`，完成后再改为 `已开发`。

## 3. 状态约定

| 状态 | 含义 |
|---|---|
| `待确认` | 已整理但尚未得到用户确认，不开始开发 |
| `待开发` | 已确认但尚未开始 |
| `开发中` | 正在实现中 |
| `已开发` | 已完成实现并做过基本验证 |
| `暂缓` | 暂不继续，等待重新排期 |

当前整份清单状态：`待开发`

## 4. P2 范围边界

本清单只覆盖 P2，包含：

1. 插件机制与自定义规则扩展。
2. 容器/K8s 诊断能力。
3. 多主机批量巡检与集中执行。
4. Web 可视化报告和更强交互界面。
5. P2 级别的安全、兼容性、测试和发布收口。

本清单不包含：

1. P0/P1 尚未完成的基础命令能力。
2. 与本地诊断无关的通用 CMDB/流程系统建设。

## 7. 当前推荐开发顺序

确认本清单后，推荐按以下大顺序推进：

1. 先做阶段 A，先把扩展协议稳定下来。
2. 再做阶段 B，落地容器/K8s 采集与诊断能力。
3. 再做阶段 C，扩展多主机并发巡检能力。
4. 再做阶段 D，补齐 Web 可视化和交互能力。
5. 最后做阶段 E，完成测试、性能和发布收口。
